СТРУКТУРА ДОКУМЕНТА

<!DOCTYPE html> <html lang="uk"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>BookMart - Магазин Книг</title> </head> <body> <main> <h2>Зображення</h2> <div> <img src="Book1.jpg" alt="Назва книги 1" width="200"> <h3>Відьмак 4</h3> <p><strong>Автор:</strong> Артем Мейсон</p> <p><strong>Ціна:</strong> 450 грн</p> <ul> <li>Фентезі</li> <li>Пригоди</li> <li>Епопея</li> </ul> <ol> <li>Тверда обкладинка</li> <li>М'яка обкладинка</li> <li>Електронний формат</li> </ol> </div> </main> </body> </html>

HTML таблиці використовуються для представлення даних у форматі рядків і стовпців, що робить їх зручними для відображення структурованої інформації, такої як таблиці з даними, статистикою чи звітами.

Основні елементи таблиць:

  • <table>: оголошує саму таблицю.
  • <tr> (table row): визначає рядок таблиці.
  • <th> (table header): визначає заголовок стовпця (зазвичай текст відображається жирним шрифтом та по центру).
  • <td> (table data): визначає клітинку з даними в рядку таблиці.

Приклад: 

                        <table border="1" cellpadding="10" cellspacing="0">
                        <thead>
                            <tr>
                                <th>Назва книги</th>
                                <th>Автор</th>
                                <th>Жанр</th>
                                <th>Ціна</th>
                            </tr>
                        </thead>
                        <tbody>
                            <tr>
                                <td>Хроніки Нарнії</td>
                                <td>Клайв Льюїс</td>
                                <td>Фентезі</td>
                                <td>500 грн</td>
                            </tr>
                        </tbody>
                    </table>

HTML дозволяє створювати списки для структурування контенту. Існує два основних типи списків: нумеровані (упорядковані) і ненумеровані (неупорядковані).

Основні елементи списків:

  • <ul>: ненумерований список (unordered list).
  • <ol>: нумерований список (ordered list).
  • <li>: пункт списку (list item).

Приклад ненумерованого списку: 

                            <ul>
                                <li>Пункт 1</li>
                                <li>Пункт 2</li>
                            </ul>

Приклад нумерованого списку: 

                            <ol>
                                <li>Перше дію</li>
                                <li>Друге дію</li>
                            </ol>

HTML-код зображень використовується для вставлення графіки в веб-сторінку. Зображення можуть бути статичними або анімованими, і вони зазвичай використовуються для ілюстрації контенту, прикраси або надання додаткової інформації.

Основні елементи зображень:

  • <img>: використовується для вставлення зображення. Основні атрибути:
    • src: вказує URL-адресу зображення.
    • alt: альтернативний текст, який відображається, якщо зображення не може бути завантажено, або використовується для доступності.

Приклад: 

                            <img src="path/to/image.jpg" alt="Опис зображення" width="300" height="200">

Десктопна версія

Десктопна версія оптимізована для екранів шириною понад 1200px.

Для десктопів використовується широка структура макета із центральним контентом, шрифтами стандартного розміру та великою кількістю простору між елементами.


            /* Десктопна версія (більше 1200px) */
            @media (min-width: 1200px) {
                header {
                    padding: 30px;
                }
            
                .student-photo {
                    max-width: 250px;
                }
            
                .screen-img {
                    max-width: 60%;
                }
            
                .sidebar {
                    width: 250px;
                }
            
                .content-section.active {
                    width: 1200px; /* Контент знову обмежується */
                }
            }

Планшетна версія

Планшетна версія оптимізована для ширини екранів від 768px до 1199px.

Основні зміни: зменшення розміру шрифтів, оптимізація відступів, перегрупування елементів для забезпечення кращої читабельності.


        /* Планшетна версія (від 768px до 1199px) */
        @media (max-width: 1199px) and (min-width: 768px) {
            header {
                padding: 25px;
            }
        
            .student-photo {
                max-width: 180px;
            }
        
            .screen-img {
                max-width: 50%;
            }
        
            .sidebar {
                width: 220px;
            }
        
            main {
                flex-direction: column;
                padding: 15px;
            }
        
            .content-section.active {
                width: 100%; /* Контент займає всю ширину */
            }
        }

Мобільна версія

Мобільна версія розроблена для ширини екранів менше 768px.

Основні зміни: вертикальне розташування елементів, максимальна адаптація навігації, приховування неважливих блоків.


            /* Мобільна версія (менше 768px) */
            @media (max-width: 767px) {
                header {
                    padding: 15px;
                    flex-direction: column;
                    align-items: center;
                }
            
                .student-photo {
                    max-width: 150px;
                    margin-top: 15px;
                }
            
                .screen-img {
                    max-width: 100%;
                    margin-top: 15px;
                }
            
                .sidebar {
                    width: 100%;
                    margin-right: 0;
                }
            
                nav {
                    flex-direction: column;
                    align-items: center;
                }
            
                .content-section.active {
                    width: 100%;
                    padding: 10px;
                }
            }

Пояснення підходу

Для адаптивної верстки використані брейкпоінти 768px та 1024px. Для кожного розміру екрану створено окремі стилі, що забезпечують оптимальний UX. Головні аспекти:

  • Великий розмір шрифтів та просторі блоки для десктопів.
  • Оптимізація відступів для планшетів.
  • Спрощення структури для мобільних пристроїв.

Логічні оператори

Логічні оператори (and, not, or) у медіа-запитах дозволяють комбінувати кілька умов для точнішої настройки стилів для різних пристроїв. Це дає можливість створювати гнучкіші та більш оптимізовані дизайни для кожного типу екрана.

Висновки до лабораторної роботи №4

  1. Розробка адаптивної верстки:

    У ході лабораторної роботи було розроблено адаптивну верстку для трьох основних типів пристроїв: десктопів, планшетів і мобільних телефонів. Використання медіа-запитів дозволило реалізувати стилі, оптимізовані для кожного розміру екрану, що забезпечило зручне використання сайту на будь-якому пристрої.

  2. Десктопна версія:

    Верстка для десктопних пристроїв була налаштована для екранів шириною понад 1200px. Було приділено увагу правильному розташуванню елементів, оптимальній ширині контейнерів та коректному розміру шрифтів для забезпечення зручного перегляду і читабельності.

  3. Планшетна версія:

    Для екранів від 768px до 1199px застосовано стилі, що дозволяють компактно розташувати елементи і забезпечити зручність навігації. Було адаптовано меню і контент для зручного перегляду на пристроях середнього розміру.

  4. Мобільна версія:

    Для пристроїв з екранами менше 768px застосовано мінімалістичний дизайн, що включає лише найнеобхідніші елементи. Це забезпечило швидке завантаження сторінки та зручне користування на малих екранах.

  5. Використання логічних операторів:

    Використання логічних операторів: Під час розробки адаптивної верстки застосовано логічні оператори and та not у медіа-запитах. Це дозволило точно налаштувати стилі для різних пристроїв, підвищивши ефективність адаптації дизайну.

  6. Висновок:

    Отримані навички з адаптивної верстки допоможуть розробляти сучасні веб-сторінки, які задовольняють потреби користувачів на різних пристроях. Використання медіа-запитів і логічних операторів забезпечує високий рівень гнучкості та ефективності дизайну, що робить сайти зручними і доступними для всіх користувачів.
